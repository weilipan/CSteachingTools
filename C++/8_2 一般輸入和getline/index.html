使用 cin >> var

string s;
cin >> s;
變數 s: ""

特性：讀取直到遇到空白 (Space, Tab, Newline) 為止。會跳過開頭的空白。

使用 getline(cin, var)

string s;
getline(cin, s);
變數 s: ""

特性：讀取直到遇到換行符號 ('\n') 為止。會讀取並捨棄換行符號。

什麼時候該用哪個？ (Best Practices)

🔹 使用 cin 的時機

  • 讀取單個單字或數值 (如 int, double)。
  • 輸入資料以空白分隔 (例如：10 20 30)。
  • 不需要包含空白的字串。

🔸 使用 getline 的時機

  • 讀取整行文字 (包含空白)。
  • 輸入姓名、地址、句子等可能包含空白的長字串。
  • 需要精確控制讀取到換行符號為止。

⚠️ 常見陷阱：混合使用

如果在 cin >> n; 之後接著使用 getline，cin 會在緩衝區留下一個換行符號 (Newline)，導致接下來的 getline 讀到空字串。

解法： 在 getline 前使用 cin.ignore(); 清除緩衝區。
