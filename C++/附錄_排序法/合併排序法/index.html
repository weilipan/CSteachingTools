🔀 C++ 合併排序法 (Merge Sort)

分而治之的經典排序演算法

📚 演算法概述

合併排序法是一種採用分而治之 (Divide and Conquer)策略的排序演算法。它將陣列遞迴地分割成更小的子陣列，直到每個子陣列只包含一個元素，然後再將這些子陣列合併成有序的陣列。

核心概念：

  • 分割 (Divide)：將陣列從中間分成兩半
  • 遞迴 (Conquer)：對兩個子陣列分別進行合併排序
  • 合併 (Merge)：將兩個已排序的子陣列合併成一個有序陣列

特性：

  • ✅ 穩定排序：相同元素的相對位置不變
  • ✅ 效率穩定：最佳、平均、最差時間複雜度都是 O(n log n)
  • ⚠️ 需要額外空間：空間複雜度為 O(n)
  • ✅ 適合大型資料：效能穩定可預測

💻 C++ 程式碼實作

#include <iostream>
#include <vector>
using namespace std;

// 合併兩個已排序的子陣列
void merge(vector<int>& arr, int left, int mid, int right) {
    int n1 = mid - left + 1;  // 左子陣列大小
    int n2 = right - mid;      // 右子陣列大小
    
    // 建立臨時陣列
    vector<int> L(n1), R(n2);
    
    // 複製資料到臨時陣列
    for (int i = 0; i < n1; i++)
        L[i] = arr[left + i];
    for (int j = 0; j < n2; j++)
        R[j] = arr[mid + 1 + j];
    
    // 合併臨時陣列回原陣列
    int i = 0;    // 左子陣列索引
    int j = 0;    // 右子陣列索引
    int k = left; // 合併後陣列索引
    
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) {
            arr[k] = L[i];
            i++;
        } else {
            arr[k] = R[j];
            j++;
        }
        k++;
    }
    
    // 複製剩餘元素（如果有）
    while (i < n1) {
        arr[k] = L[i];
        i++;
        k++;
    }
    
    while (j < n2) {
        arr[k] = R[j];
        j++;
        k++;
    }
}

// 合併排序主函式
void mergeSort(vector<int>& arr, int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;
        
        // 遞迴排序左半部
        mergeSort(arr, left, mid);
        
        // 遞迴排序右半部
        mergeSort(arr, mid + 1, right);
        
        // 合併兩個已排序的半部
        merge(arr, left, mid, right);
    }
}

int main() {
    vector<int> arr = {38, 27, 43, 3, 9, 82, 10};
    int n = arr.size();
    
    cout << "排序前: ";
    for (int num : arr) cout << num << " ";
    cout << endl;
    
    mergeSort(arr, 0, n - 1);
    
    cout << "排序後: ";
    for (int num : arr) cout << num << " ";
    cout << endl;
    
    return 0;
}

⏱️ 時間複雜度分析

1️⃣ 最佳情況 (Best Case)：$O(n \log n)$

即使陣列已經排序，合併排序仍需要進行所有的分割和合併操作。

2️⃣ 平均情況 (Average Case)：$O(n \log n)$

無論資料如何分布，時間複雜度都保持穩定。

3️⃣ 最差情況 (Worst Case)：$O(n \log n)$

合併排序的效能不受輸入資料影響，始終保持 $O(n \log n)$。

📊 詳細計算過程

步驟 1：分析遞迴樹的結構

假設陣列大小為 $n$，合併排序會將陣列不斷對半分割：

  • 第 0 層：1 個大小為 $n$ 的問題
  • 第 1 層：2 個大小為 $\frac{n}{2}$ 的問題
  • 第 2 層：4 個大小為 $\frac{n}{4}$ 的問題
  • 第 k 層：$2^k$ 個大小為 $\frac{n}{2^k}$ 的問題

步驟 2：計算遞迴樹的高度

當子問題大小為 1 時停止分割：

$$\frac{n}{2^h} = 1 \implies 2^h = n \implies h = \log_2 n$$

因此，遞迴樹的高度為 $\log_2 n$（共有 $\log_2 n + 1$ 層）。

步驟 3：計算每層的工作量

在第 $k$ 層：

  • 有 $2^k$ 個子問題
  • 每個子問題大小為 $\frac{n}{2^k}$
  • 合併一個大小為 $m$ 的子陣列需要 $O(m)$ 時間

因此，第 $k$ 層的總工作量為：

$$T_k = 2^k \times \frac{n}{2^k} = n$$

步驟 4：計算總時間複雜度

總共有 $\log_2 n + 1$ 層，每層工作量都是 $n$：

$$T(n) = n \times (\log_2 n + 1) = n \log_2 n + n$$

使用大 O 記號，忽略低階項和常數：

$$T(n) = O(n \log n)$$

步驟 5：遞迴關係式驗證

合併排序的遞迴關係式為：

$$T(n) = 2T\left(\frac{n}{2}\right) + O(n)$$

其中：

  • $2T\left(\frac{n}{2}\right)$：兩個子問題的時間
  • $O(n)$：合併兩個子陣列的時間

使用主定理 (Master Theorem)：$a=2, b=2, f(n)=n$

$$n^{\log_b a} = n^{\log_2 2} = n^1 = n$$

因為 $f(n) = \Theta(n^{\log_b a})$，所以：

$$T(n) = \Theta(n \log n)$$

實際範例計算

假設 $n = 8$：

  • 層數：$\log_2 8 = 3$（共 4 層：0, 1, 2, 3）
  • 第 0 層：1 個大小 8 的問題，工作量 = 8
  • 第 1 層：2 個大小 4 的問題，工作量 = 2×4 = 8
  • 第 2 層：4 個大小 2 的問題，工作量 = 4×2 = 8
  • 第 3 層：8 個大小 1 的問題，工作量 = 8×1 = 8
  • 總工作量：$8 \times 4 = 32 = 8 \times \log_2 8 + 8$

💾 空間複雜度分析

空間複雜度：$O(n)$

📊 詳細計算過程

步驟 1：識別空間使用來源

合併排序需要額外空間的地方：

  • 臨時陣列：用於合併操作
  • 遞迴呼叫堆疊：用於儲存遞迴狀態

步驟 2：分析臨時陣列空間

在合併函式中，我們建立兩個臨時陣列 L 和 R：

  • L 陣列大小：$mid - left + 1$
  • R 陣列大小：$right - mid$
  • 總大小：$(mid - left + 1) + (right - mid) = right - left + 1$

在最頂層的合併中，$left = 0, right = n-1$：

$$\text{臨時陣列大小} = (n-1) - 0 + 1 = n$$

步驟 3：分析遞迴堆疊空間

遞迴深度等於遞迴樹的高度：

$$\text{遞迴深度} = \log_2 n$$

每層遞迴需要儲存：

  • 函式參數：left, right, mid（常數空間）
  • 區域變數：索引變數等（常數空間）

因此，遞迴堆疊空間為：

$$O(\log n)$$

步驟 4：計算總空間複雜度

總空間使用量為：

$$S(n) = \underbrace{O(n)}_{\text{臨時陣列}} + \underbrace{O(\log n)}_{\text{遞迴堆疊}}$$

取主導項：

$$S(n) = O(n)$$

步驟 5：優化考量

方法 1：重複使用臨時陣列

可以在最開始分配一個大小為 $n$ 的臨時陣列，在所有合併操作中重複使用：

  • 空間複雜度仍為 $O(n)$
  • 但減少了記憶體分配次數，提升效能

方法 2：原地合併（理論）

雖然存在原地合併演算法（空間複雜度 $O(1)$），但：

  • 實作複雜度大幅增加
  • 時間複雜度會增加到 $O(n^2 \log n)$
  • 失去實用價值

實際範例計算

假設 $n = 16$：

  • 臨時陣列：最多需要 16 個整數的空間
  • 遞迴深度：$\log_2 16 = 4$ 層
  • 每層堆疊：約 3-4 個整數變數
  • 遞迴堆疊總計：約 $4 \times 4 = 16$ 個整數
  • 總空間：約 $16 + 16 = 32$ 個整數 = $O(n)$

與其他排序演算法比較

演算法 時間複雜度 空間複雜度 穩定性
合併排序 $O(n \log n)$ $O(n)$ ✅ 穩定
快速排序 $O(n \log n)$ 平均 $O(\log n)$ ❌ 不穩定
堆積排序 $O(n \log n)$ $O(1)$ ❌ 不穩定
插入排序 $O(n^2)$ $O(1)$ ✅ 穩定

📝 演算法步驟詳解

步驟 1：分割 (Divide)

計算中間點：$mid = left + \frac{right - left}{2}$

將陣列分成兩個子陣列：

  • 左子陣列：$[left, mid]$
  • 右子陣列：$[mid+1, right]$

步驟 2：遞迴排序 (Conquer)

對左右兩個子陣列分別遞迴呼叫合併排序：

  • mergeSort(arr, left, mid)
  • mergeSort(arr, mid+1, right)

遞迴終止條件：$left \geq right$（子陣列只有一個或零個元素）

步驟 3：合併 (Merge)

將兩個已排序的子陣列合併成一個有序陣列：

  1. 建立臨時陣列 L 和 R，複製左右子陣列的資料
  2. 使用三個指標 i, j, k：
    • i：指向 L 陣列當前元素
    • j：指向 R 陣列當前元素
    • k：指向原陣列當前位置
  3. 比較 L[i] 和 R[j]，將較小者放入 arr[k]
  4. 移動相應的指標
  5. 重複直到其中一個陣列處理完畢
  6. 將剩餘元素複製回原陣列

🔍 執行追蹤範例

以陣列 [38, 27, 43, 3, 9, 82, 10] 為例：

初始陣列: [38, 27, 43, 3, 9, 82, 10]

分割階段：
[38, 27, 43, 3, 9, 82, 10]
    ↓
[38, 27, 43, 3] | [9, 82, 10]
    ↓                  ↓
[38, 27] | [43, 3]   [9, 82] | [10]
    ↓         ↓         ↓        ↓
[38] | [27] [43] | [3] [9] | [82] [10]

合併階段：
[38] | [27] → [27, 38]
[43] | [3]  → [3, 43]
[27, 38] | [3, 43] → [3, 27, 38, 43]

[9] | [82] → [9, 82]
[9, 82] | [10] → [9, 10, 82]

[3, 27, 38, 43] | [9, 10, 82] → [3, 9, 10, 27, 38, 43, 82]

最終結果: [3, 9, 10, 27, 38, 43, 82]

⚖️ 優缺點分析

✅ 優點

  • 穩定的效能：無論資料如何分布，時間複雜度都是 $O(n \log n)$
  • 穩定排序：保持相同元素的相對順序
  • 可預測性：效能不受輸入影響
  • 適合外部排序：適用於處理大型檔案
  • 平行化友善：容易改寫成平行演算法
  • 適合鏈結串列：不需要隨機存取

❌ 缺點

  • 需要額外空間：空間複雜度 $O(n)$
  • 不是原地排序：需要額外的記憶體
  • 小型陣列效率較低：常數因子較大
  • 遞迴開銷：函式呼叫有額外成本
  • 快取效能：記憶體存取模式不如快速排序

🎯 應用場景

適合使用合併排序的情況：

  • 🔹 需要穩定排序的場景
  • 🔹 資料量大且需要可預測的效能
  • 🔹 外部排序（資料存在磁碟）
  • 🔹 排序鏈結串列
  • 🔹 需要平行處理的排序任務
  • 🔹 對最差情況效能有嚴格要求

不適合使用的情況：

  • 🔸 記憶體受限的環境（考慮使用堆積排序）
  • 🔸 小型陣列（考慮使用插入排序）
  • 🔸 需要原地排序（考慮使用快速排序或堆積排序）
