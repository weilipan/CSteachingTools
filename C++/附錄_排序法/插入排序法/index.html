C++ 插入排序法 (Insertion Sort)

逐步視覺化與演算法解析

準備就緒

C++ 實作程式碼

void insertionSort(std::vector<int>& arr) {
    int n = arr.size();
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;

        // 將大於 key 的元素往後移動
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}

程式碼解析：

  • std::vector<int>& arr: 使用 C++ 的 vector 容器，並以傳參考 (Reference) 方式傳遞，避免複製陣列。
  • int key = arr[i]: 暫存當前要插入的元素。
  • while (j >= 0 && arr[j] > key): 這是插入排序的核心，將比 key 大的元素不斷往後移，騰出空間。
  • arr[j + 1] = key: 將 key 放入正確的位置。

演算法說明

運作原理

插入排序法 (Insertion Sort) 的運作方式類似於整理撲克牌。我們將陣列分為「已排序」與「未排序」兩部分。 每次從未排序的部分取出一個元素，將其插入到已排序部分的適當位置，直到所有元素都排序完成。

  • 步驟 1： 從第二個元素開始（假設第一個元素已排序）。
  • 步驟 2： 將當前元素與已排序部分的元素從後往前比較。
  • 步驟 3： 若已排序元素大於當前元素，則將該元素往後移。
  • 步驟 4： 重複步驟 3，直到找到適當位置或比較完所有已排序元素。
  • 步驟 5： 將當前元素插入該位置。

複雜度分析

時間複雜度 (Time Complexity)

  • 最佳情況 (Best Case): $O(n)$
    當陣列已經是排序好的時候，只需比較 $n-1$ 次。
  • 平均情況 (Average Case): $O(n^2)$
    平均需要比較和移動 $\frac{n^2}{4}$ 次。
  • 最差情況 (Worst Case): $O(n^2)$
    當陣列是反向排序時，每次插入都需要移動所有已排序元素。

空間複雜度 (Space Complexity)

  • 空間複雜度: $O(1)$
    這是一個原地排序演算法 (In-place Algorithm)，只需要一個額外的暫存空間。

詳細計算方式

假設陣列長度為 $n$。

在第 $i$ 回合 ($i$ 從 1 到 $n-1$)，我們將第 $i$ 個元素插入到前 $i$ 個已排序的元素中。 在最差情況下，第 $i$ 個元素需要與前 $i$ 個元素進行比較並移動。

總比較次數為： $$ \sum_{i=1}^{n-1} i = 1 + 2 + ... + (n-1) = \frac{n(n-1)}{2} \approx \frac{n^2}{2} $$

因此，時間複雜度為 $O(n^2)$。
